# Contributing to OpenSecurity

Thanks for your interest in contributing.

## Development Setup

```bash
npm install
npm run build
```

## Common Commands

```bash
npm test
npm run lint
npm run build
npm run dev -- scan --dry-run
```

## Pull Requests

- Keep changes focused and scoped.
- Add or update tests for new behavior.
- Run `npm test`, `npm run lint`, and `npm run build` before opening a PR.
- Do not add or log real secrets.

## Reporting Bugs

Open an issue with:

- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Node version)

## Security

If you discover a security issue, avoid disclosing it publicly. Prefer a private GitHub Security Advisory or a minimal public issue requesting private follow‑up.
